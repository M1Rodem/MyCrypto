# CryptoAPI - Composite Encryption Service

## Описание проекта

CryptoAPI - RESTful веб-сервис на ASP.NET Core для шифрования и дешифрования текстовых данных с использованием композитного алгоритма. Сервис объединяет несколько криптографических методов для обеспечения многоуровневой защиты данных.

## Основные возможности

- Шифрование текста с использованием пользовательского ключа
- Дешифрование зашифрованных данных
- Генерация криптостойких случайных ключей
- Автоматическое сохранение ключа в защищенной HTTP-куке
- Защита от timing-атак
- Автоматическая очистка чувствительных данных из памяти

## Архитектура композитного шифрования

```
Input: Text + Password
        |
        v
[1] PBKDF2 (1,000,000 итераций, SHA-512)
    - Выход: 64 байта
    - Первые 32 байта: Encryption Key
    - Последние 32 байта: MAC Key
        |
        v
[2] CustomCryptoService.Transform()
    - Псевдослучайный XOR (LCG)
    - Аддитивный сдвиг (mod 17)
    - Блочная перестановка (Reverse блоков по 4 байта)
        |
        v
[3] ChaCha20 Stream Cipher
    - Nonce: 8 байт
    - Шифрование результата предыдущего шага
        |
        v
[4] Формирование выходной структуры:
    [Version:1] [Salt:16] [Nonce:8] [EncryptedData] [HMAC-SHA256:32]
        |
        v
Output: Base64 Encoded String
```

## Технологический стек

- .NET 6.0 / .NET 8.0
- ASP.NET Core Web API
- BouncyCastle.Crypto (ChaCha20)
- System.Security.Cryptography (PBKDF2, HMAC-SHA256, RNGCryptoServiceProvider)

## Механизм работы с ключами

### Приоритет получения ключа:
1. Ключ из тела запроса (поле `key`)
2. Ключ из HTTP-куки `crypto_key` (если не передан в запросе)

### Cookie настройки:
- `HttpOnly: true` - защита от XSS
- `Secure: true` - только по HTTPS
- `SameSite: Strict` - защита от CSRF
- `Expires: 30 минут` - ограниченное время жизни

## Детали реализации

### PBKDF2 настройки
- Алгоритм: SHA-512
- Итерации: 1,000,000
- Выход: 64 байта (512 бит)
- Соль: 16 случайных байт

### Пользовательский алгоритм (CustomCryptoService)
- Генератор псевдослучайных чисел: LCG (Linear Congruential Generator)
- Формула: `seed = (seed * 1103515245 + 12345) & int.MaxValue`
- Операции:
  - XOR с псевдослучайным байтом
  - Аддитивный сдвиг на `(rnd % 17)`
  - Перестановка блоков (reverse каждого блока по 4 байта)

### ChaCha20 настройки
- Nonce размер: 8 байт (64 бита)
- Библиотека: BouncyCastle

### HMAC настройки
- Алгоритм: HMAC-SHA256
- Размер выхода: 32 байта
- Назначение: проверка целостности и подлинности

## Безопасность

### Реализованные меры:
1. Timing-safe сравнение HMAC (`CryptographicOperations.FixedTimeEquals`)
2. ZeroMemory для чувствительных данных
3. HttpOnly cookies для защиты ключа
4. PBKDF2 с высокой итерацией для защиты паролей
5. Разделение ключей (Encryption Key / MAC Key)

### Важные замечания:
- Пользовательский алгоритм является дополнительным слоем обфускации
- Основная криптографическая стойкость обеспечивается ChaCha20 и HMAC
- Ключи не логируются и не сохраняются на сервере

## Установка и запуск

### Требования
- .NET 6.0 SDK или выше
- NuGet пакет: BouncyCastle.Cryptography

### Установка зависимостей

Сервер запустится на `https://localhost:5001` и `http://localhost:5000`

## Формат зашифрованных данных

Каждый зашифрованный результат имеет следующую структуру:

| Поле | Размер (байт) | Описание |
|------|--------------|----------|
| Version | 1 | Версия алгоритма (текущая: 1) |
| Salt | 16 | Соль для PBKDF2 |
| Nonce | 8 | Nonce для ChaCha20 |
| Encrypted Data | Переменный | Зашифрованные данные |
| HMAC | 32 | HMAC-SHA256 от всех предыдущих полей |
